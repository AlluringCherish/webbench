Tester Feedback for MusicStreaming Application

---- Backend Issues ----
1. Data Loading:
- The backend functions correctly parse and load data from text files using the correct delimiters and handle missing files gracefully by returning empty lists.

2. Routes & Functionality:
- The root redirection correctly sends users from '/' to '/dashboard'.
- Dashboard page implementation correctly generates featured songs (top 5 by play counts) and trending artists (top 5 by total plays), matching design spec.
- Song catalog page properly filters songs by search query and genre, with context variables matching specifications.
- Song details page correctly returns 404 on invalid song_id and supplies all required song details and artist profile URL.
- Playlists pages: listing, creation (with input validation), details, deletion, and song removal functionality are implemented as required.
- Album browsing and detail pages support search, sorting by title/year, and display correct album/song info.
- Adding an album to playlist posts to correct route and updates playlist total songs.
- Artists page supports search and sorting as specified.
- Artist profile page displays artist info and songs.
- Genre exploration page works with genre selection and displays songs/artists accordingly.

- No major backend errors or crashes noticed based on code review.

---- Frontend Issues ----
1. General:
- All pages have appropriate container divs with specified IDs, consistent with design spec.

2. Dashboard.html:
- Page title and header "Music Streaming Dashboard" present.
- Element IDs 'dashboard-page', 'featured-songs', and buttons with IDs 'browse-songs-button', 'my-playlists-button', and 'trending-artists-button' exist and map correctly to routes.
