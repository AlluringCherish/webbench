--- Backend Issues ---
1. In 'app.py', POST /playlists/create attempts to read form keys 'playlist-name-input' and 'playlist-description-input',
   but in 'create_playlist.html', input fields use 'name="playlist_name"' and 'name="playlist_description"',
   causing playlist creation to fail due to missing title and server-side validation error.
2. Missing implementation for POST route '/songs/<int:song_id>/add_to_playlist' or similar to add a single song to playlist.
3. In 'album_details.html', the form for adding album songs to a playlist lacks a playlist selection input (playlist_id),
   but backend expects a POST field 'playlist_id', resulting in an error 400 on submission.
4. Some routes returning 404 as test environment can't instantiate Flask app fully; would require integration testing live.

--- Frontend Issues ---

   Inputs have name='playlist_name' and 'playlist_description' but backend looks for 'playlist-name-input' and 'playlist-description-input'.
2. Missing playlist selection input in album_details.html form for adding album songs to playlist.
3. 'Add to Playlist' buttons on song_catalog.html (IDs 'add-to-playlist-button-{song_id}') post to non-existent backend route,
   making them non-functional.

5. Navigation buttons link correctly except where backend routes are missing.
6. No significant UI inconsistencies; styling and responsiveness not fully verifiable.
