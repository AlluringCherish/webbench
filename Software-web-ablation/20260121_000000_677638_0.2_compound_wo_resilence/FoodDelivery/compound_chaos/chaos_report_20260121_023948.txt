         ## Section 3: Data Schemas
         
         ### 1. Restaurants Data File
         - File Name: data/restaurants.txt
         - Fields (pipe-separated):
           - restaurant_id (int)
           - name (string)
           - cuisine (string)
           - address (string)
           - phone (string)
           - rating (float)
           - delivery_time (int - minutes)
           - min_order (float)
         - Purpose: Stores restaurant details including identification, contact info, ratings, and delivery info.
         - Example Data Rows:
           ```
           1|Dragon House|Chinese|123 Main St|555-0001|4.5|30|15.00
           2|La Bella Italia|Italian|456 Oak Ave|555-0002|4.8|25|20.00
           3|Taj Mahal|Indian|789 Elm St|555-0003|4.6|35|18.00
           ```
         
         ---
         
         ### 2. Menus Data File
         - File Name: data/menus.txt
         - Fields (pipe-separated):
           - item_id (int)
           - restaurant_id (int)
           - item_name (string)
           - category (string)
           - description (string)
           - price (float)
           - availability (int - 1=available, 0=unavailable)
         - Purpose: Stores all menu items per restaurant with descriptions and pricing.
         - Example Data Rows:
           ```
           1|1|Fried Rice|Main Course|Steamed rice with vegetables and egg|12.99|1
           2|1|Spring Rolls|Appetizer|Crispy rolls with pork filling|8.99|1
           3|2|Spaghetti Carbonara|Pasta|Classic Italian pasta with cream sauce|14.99|1
           ```
         
         ---
         
         ### 3. Cart Data File
         - File Name: data/cart.txt
         - Fields (pipe-separated):
           - cart_id (int)
           - item_id (int)
           - restaurant_id (int)
           - quantity (int)
           - added_date (string - ISO date, YYYY-MM-DD)
         - Purpose: Stores shopping cart items for users (no authentication, so per session or global cart assumed).
         - Example Data Rows:
           ```
           1|1|1|2|2025-01-15
           2|3|2|1|2025-01-16
           ```
         
         ---
         
         ### 4. Orders Data File
         - File Name: data/orders.txt
         - Fields (pipe-separated):
           - order_id (int)
           - customer_name (string)
           - restaurant_id (int)
           - order_date (string - ISO date, YYYY-MM-DD)
           - total_amount (float)
           - status (string) (e.g. Delivered, On the Way, Preparing)
           - delivery_address (string)
           - phone_number (string)
         - Purpose: Records completed and active orders.
         - Example Data Rows:
           ```
           1|John Doe|1|2025-01-10|21.98|Delivered|123 Main St, NYC|555-1234
           2|Jane Smith|2|2025-01-14|14.99|On the Way|456 Oak Ave, LA|555-5678
           ```
         
         ---
         
         ### 5. Order Items Data File
         - File Name: data/order_items.txt
         - Fields (pipe-separated):
           - order_item_id (int)
           - order_id (int)
           - item_id (int)
           - quantity (int)
           - price (float)
         - Purpose: Stores each item ordered for each order with quantity and price details.
         - Example Data Rows:
           ```
           1|1|1|2|12.99
           2|1|2|1|8.99
           3|2|3|1|14.99
           ```
         
         ---
         
         ### 6. Deliveries Data File
         - File Name: data/deliveries.txt
         - Fields (pipe-separated):
           - delivery_id (int)
           - order_id (int)
           - driver_name (string)
           - driver_phone (string)
           - vehicle_info (string)
           - status (string) (e.g. Delivered, On the Way)
           - estimated_time (string - datetime ISO format, YYYY-MM-DD HH:MM)
         - Purpose: Tracks delivery details and statuses.
         - Example Data Rows:
           ```
           1|1|Mike Johnson|555-9001|Bike|Delivered|2025-01-10 18:45
           2|2|Sarah Williams|555-9002|Car|On the Way|2025-01-14 19:30

           - rating (int - 1 to 5 stars)
           - review_text (string)
           - review_date (string - ISO date, YYYY-MM-DD)

           1|1|Alice Johnson|5|Excellent food and fast delivery!|2025-01-12
           2|2|Bob Williams|4|Great pasta, slightly delayed delivery.|2025-01-13
