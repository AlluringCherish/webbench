         # SmartHomeManager Design Specification
         
         ---
         
         ## Section 1: Flask Routes Specification
         
         | Endpoint URL                | HTTP Methods | Function Name           | Template Rendered          | Context Variables (Name: Type)                          |
         |-----------------------------|--------------|------------------------|----------------------------|--------------------------------------------------------|
         | /
         | GET        | root_redirect          | *Redirects to /dashboard*| None                                                   |
         | /dashboard                 | GET          | dashboard              | dashboard.html             | devices: list, rooms: list, device_counts: dict         |
         | /devices                   | GET          | device_list            | device_list.html           | devices: list                                             |
         | /devices/add               | GET, POST   | add_device             | add_device.html            | device_types: list[str], rooms: list[str], errors: dict (optional) |
         | /device/<int:device_id>    | GET, POST   | device_control         | device_control.html        | device: dict, status: str                                 |
         | /automation                | GET, POST   | automation_rules       | automation_rules.html      | rules: list, devices: list[str], errors: dict (optional) |
         | /energy                   | GET, POST   | energy_report          | energy_report.html         | energy_logs: list, total_consumption: float, total_cost: float, filter_date: str (optional) |
         | /activity                 | GET, POST   | activity_logs          | activity_logs.html         | activities: list, search_query: str (optional)           |
         | /device/<int:device_id>/control | POST         | control_device_action  | None (Redirect)            | None                                                    |
         
         ---
         
         ### Explanation of Routes:
         
         - **/** : Root redirect route to the dashboard page.
         - **/dashboard** : Displays main dashboard overview with device summaries and room list.
         - **/devices** : Lists all devices with quick control buttons.
         - **/devices/add** : Form page to add a new device, handles GET for form display and POST for submission.
         - **/device/<device_id>** : Device control page for a specific device, GET to display, POST to save control changes.
         - **/automation** : Automation rules management page, GET to display, POST to add new rule.
         - **/energy** : Energy report page, GET to display all or POST to apply date filter.
         - **/activity** : Activity logs page, GET to display all or POST to apply search filters.
         - **/device/<device_id>/control** : POST route to handle device control toggles submitted from device list quick control buttons.

         ## Section 3: Data File Schemas
         
         ---
         
         ### 1. users.txt
         - Path: data/users.txt
         - Fields (pipe-delimited):
           1. username (str) - Username identifier
           2. email (str) - User email address
         
         - Example Lines:
           ```
           john_doe|john@example.com
           jane_smith|jane@example.com
           ```
         
         ---
         
         ### 2. devices.txt
         - Path: data/devices.txt
         - Fields (pipe-delimited):
           1. username (str) - Owner username
           2. device_id (int) - Unique device identifier
           3. device_name (str) - Device display name
           4. device_type (str) - Type of device (Light, Thermostat, etc.)
           5. room (str) - Room location
           6. brand (str) - Device brand
           7. model (str) - Device model
           8. status (str) - Device status (Online/Offline)
           9. power (str) - Power state (on/off)
           10. brightness (int or empty) - Brightness level (0-100) or empty if N/A
           11. temperature (int or empty) - Temperature setting or empty if N/A
           12. mode (str) - Mode setting (e.g., Auto, Manual, empty)
           13. schedule_time (str) - Scheduled time string or empty
         
         - Example Lines:
           ```
           john_doe|1|Living Room Light|Light|Living Room|Philips|Hue|Online|on|75|||Auto|
           john_doe|2|Bedroom Thermostat|Thermostat|Bedroom|Nest|Learning|Online|on||72|Auto|22:00
           jane_smith|3|Kitchen Camera|Camera|Kitchen|Ring|Indoor|Online|on||||Manual|
           ```
         
         ---
         
         ### 3. rooms.txt
         - Path: data/rooms.txt
         - Fields (pipe-delimited):
           1. username (str) - Owner username
           2. room_id (int) - Unique room identifier
           3. room_name (str) - Name of the room
         
         - Example Lines:
           ```
           john_doe|1|Living Room
           john_doe|2|Bedroom
           john_doe|3|Kitchen
           jane_smith|1|Living Room
           ```
         
         ---
         
         ### 4. automation_rules.txt
         - Path: data/automation_rules.txt
         - Fields (pipe-delimited):
           1. username (str) - Owner username
           2. rule_id (int) - Unique automation rule identifier
           3. rule_name (str) - Name of the automation rule
           4. trigger_type (str) - Trigger type (Time, Motion, Temperature)
           5. trigger_value (str) - Trigger value (e.g., time string, detected)
           6. action_device_id (int) - Target device ID for the action
           7. action_type (str) - Action type (Turn On, Turn Off, Set Brightness, Set Temperature)
           8. action_value (str) - Value for the action or empty
           9. enabled (bool as str) - 'true' or 'false' indicating rule active
         
         - Example Lines:
           ```
           john_doe|1|Morning Lights|Time|07:00|1|Turn On||true
           john_doe|2|Night Mode|Time|22:00|1|Set Brightness|20|true
           jane_smith|1|Motion Alert|Motion|detected|3|Turn On||true
           ```
         
         ---
         
         ### 5. energy_logs.txt
         - Path: data/energy_logs.txt
         - Fields (pipe-delimited):
           1. username (str) - Owner username
           2. device_id (int) - Device identifier
           3. date (str, YYYY-MM-DD) - Date of consumption record
           4. consumption_kwh (float) - Energy consumption in kWh
         
         - Example Lines:
           ```
           john_doe|1|2024-11-01|0.5
           john_doe|2|2024-11-01|2.3
           john_doe|1|2024-11-02|0.6
           jane_smith|3|2024-11-01|0.2
           ```
         
         ---
         
         ### 6. activity_logs.txt
         - Path: data/activity_logs.txt
         - Fields (pipe-delimited):
           1. username (str) - Owner username
           2. timestamp (str, YYYY-MM-DD HH:MM:SS) - Timestamp of the activity
           3. device_id (int) - Id of the device
           4. action (str) - Action performed
           5. details (str) - Additional details about the action
         
         - Example Lines:
           ```
           john_doe|2024-11-01 07:00:00|1|Power On|Automation triggered: Morning Lights
           john_doe|2024-11-01 08:30:00|2|Settings Changed|Temperature set to 72
           jane_smith|2024-11-01 09:15:00|3|Power On|Manual control
           ```

         ## Section 3: Data File Schemas
         
         ### 1. users.txt
         - Path: data/users.txt
         - Fields (pipe-delimited):
           1. username (str) - User's unique username
           2. email (str) - User's email address
         
         - Example Lines:
           ```
           john_doe|john@example.com
           jane_smith|jane@example.com
           ```
         
         ---
         
         ### 2. devices.txt
         - Path: data/devices.txt
         - Fields (pipe-delimited):
           1. username (str) - Owning user's username
           2. device_id (int) - Unique device identifier
           3. device_name (str) - Device display name
           4. device_type (str) - Device type (e.g., Light, Thermostat)
           5. room (str) - Room name where device is located
           6. brand (str) - Device brand
           7. model (str) - Device model
           8. status (str) - Device status (Online/Offline)
           9. power (str) - Power status (on/off)
           10. brightness (int or empty) - Brightness level (0-100) or empty if not applicable
           11. temperature (int or empty) - Temperature setting or empty if not applicable
           12. mode (str) - Mode setting (Auto, Manual, or empty)
           13. schedule_time (str) - Scheduled time or empty
         
         - Example Lines:
           ```
           john_doe|1|Living Room Light|Light|Living Room|Philips|Hue|Online|on|75|||Auto|
           john_doe|2|Bedroom Thermostat|Thermostat|Bedroom|Nest|Learning|Online|on||72|Auto|22:00
           jane_smith|3|Kitchen Camera|Camera|Kitchen|Ring|Indoor|Online|on||||Manual|
           ```
         
         ---
         
         ### 3. rooms.txt
         - Path: data/rooms.txt
         - Fields (pipe-delimited):
           1. username (str) - Owning user's username
           2. room_id (int) - Unique room identifier
           3. room_name (str) - Name of the room
         
         - Example Lines:
           ```
           john_doe|1|Living Room
           john_doe|2|Bedroom
           john_doe|3|Kitchen
           jane_smith|1|Living Room
           ```
         
         ---
         
         ### 4. automation_rules.txt
         - Path: data/automation_rules.txt
         - Fields (pipe-delimited):
           1. username (str) - Owning user's username
           2. rule_id (int) - Unique rule identifier
           3. rule_name (str) - Name of the rule
           4. trigger_type (str) - Trigger type (Time, Motion, Temperature)
           5. trigger_value (str) - Trigger value (e.g., time or motion detected)
           6. action_device_id (int) - ID of target device
           7. action_type (str) - Action type (Turn On, Turn Off, Set Brightness, Set Temperature)
           8. action_value (str) - Value for the action or empty
           9. enabled (str) - 'true' or 'false' for rule status
         
         - Example Lines:
           ```
           john_doe|1|Morning Lights|Time|07:00|1|Turn On||true
           john_doe|2|Night Mode|Time|22:00|1|Set Brightness|20|true
           jane_smith|1|Motion Alert|Motion|detected|3|Turn On||true
           ```
         
         ---
         
         ### 5. energy_logs.txt
         - Path: data/energy_logs.txt
         - Fields (pipe-delimited):
           1. username (str) - Owning user's username
           2. device_id (int) - Device identifier
           3. date (str) - Date in YYYY-MM-DD format
           4. consumption_kwh (float) - Energy consumption in kilowatt-hours
         
         - Example Lines:
           ```
           john_doe|1|2024-11-01|0.5
           john_doe|2|2024-11-01|2.3
           john_doe|1|2024-11-02|0.6
           jane_smith|3|2024-11-01|0.2
           ```
         
         ---
         
         ### 6. activity_logs.txt
         - Path: data/activity_logs.txt
         - Fields (pipe-delimited):
           1. username (str) - Owning user's username
           2. timestamp (str) - Timestamp in YYYY-MM-DD HH:MM:SS format
           3. device_id (int) - Device ID
           4. action (str) - Action performed
           5. details (str) - Additional details
         
         - Example Lines:
           ```
           john_doe|2024-11-01 07:00:00|1|Power On|Automation triggered: Morning Lights
           john_doe|2024-11-01 08:30:00|2|Settings Changed|Temperature set to 72
           jane_smith|2024-11-01 09:15:00|3|Power On|Manual control
           ```
