         ## 4. Data Storage
         
         The 'GymMembership' application will store data locally in text files organized in the directory 'data'. The following data formats and examples are defined:
         
         ### 1. Memberships Data
         - **File Name**: `memberships.txt`
         - **Data Format**:
           ```
           membership_id|plan_name|price|billing_cycle|features|max_classes
           ```
         - **Example Data**:
           ```
           1|Basic|29.99|monthly|Gym access, 2 classes per week|8
           2|Premium|59.99|monthly|Gym access, 5 classes per week, 2 PT sessions|20
           3|Elite|99.99|monthly|Unlimited gym access, unlimited classes, 4 PT sessions, nutrition coaching|unlimited
           ```
         
         ### 2. Classes Data
         - **File Name**: `classes.txt`
         - **Data Format**:
           ```
           class_id|class_name|trainer_id|class_type|schedule_day|schedule_time|capacity|duration
           ```
         - **Example Data**:
           ```
           1|Morning Yoga|1|Yoga|Monday|06:00|20|60
           2|CrossFit Bootcamp|2|CrossFit|Tuesday|18:00|15|45
           3|Pilates Core|3|Pilates|Wednesday|10:00|18|50
           ```
         
         ### 3. Trainers Data
         - **File Name**: `trainers.txt`
         - **Data Format**:
           ```
           trainer_id|name|specialty|certifications|experience_years|bio
           ```
         - **Example Data**:
           ```
           1|Sarah Johnson|Yoga & Flexibility|Certified Yoga Instructor, CPR|8|Expert in mind-body wellness and injury prevention
           2|Mike Thompson|Strength & Conditioning|NASM-CPT, CrossFit Level 2|10|Specializes in powerlifting and functional fitness
           3|Emma Davis|Pilates|Pilates Method Alliance Certified|6|Focus on core strength and postural alignment
           ```
         
         ### 4. Bookings Data
         - **File Name**: `bookings.txt`
         - **Data Format**:
           ```
           booking_id|member_name|trainer_id|booking_date|booking_time|duration_minutes|status
           ```
         - **Example Data**:
           ```
           1|John Doe|1|2025-01-20|10:00|60|Confirmed

         - **File Name**: `memberships.txt`
         - **Data Format**:
           ```
           membership_id|plan_name|price|billing_cycle|features|max_classes

           ```
           1|Basic|29.99|monthly|Gym access, 2 classes per week|8
           2|Premium|59.99|monthly|Gym access, 5 classes per week, 2 PT sessions|20
           3|Elite|99.99|monthly|Unlimited gym access, unlimited classes, 4 PT sessions, nutrition coaching|unlimited

         - **File Name**: `classes.txt`
         - **Data Format**:
           ```
           class_id|class_name|trainer_id|class_type|schedule_day|schedule_time|capacity|duration

         - **Example Data**:
           ```
           1|Morning Yoga|1|Yoga|Monday|06:00|20|60

           ```
         
         ### 4. Bookings Data
         - **File Name**: `bookings.txt`
         - **Data Format**:
           ```
           booking_id|member_name|trainer_id|booking_date|booking_time|duration_minutes|status

         - **Template**: `dashboard.html`
         - **Context variables**:
           - `memberships`: list of membership dicts with keys: `membership_id` (int), `plan_name` (str), `price` (float), `billing_cycle` (str), `features` (str), `max_classes` (int)
           - `trainers`: list of trainer dicts with keys: `trainer_id` (int), `name` (str), `specialty` (str), `certifications` (list of str), `experience_years` (int), `bio` (str)
         
         ### 2. Route: `/memberships`
         - **Methods**: GET
         - **Description**: Page listing all membership plans with links to their details.
         - **Template**: `memberships.html`
         - **Context variables**:
           - `memberships`: list of membership dicts (same structure as above)
         
         ### 3. Route: `/plan/<int:plan_id>`
         - **Methods**: GET
         - **Description**: Page showing detailed information for a single membership plan.
         - **Template**: `plan_details.html`
         - **Context variables**:
           - `plan`: dict representing a membership plan with keys as above
         - **Behavior**:
           - If plan_id not found, redirects to `/memberships`
         
         
         ## Section 2: HTML Template Specifications
         
         ### Template: `dashboard.html`
         - **Page Title**: "Dashboard | GymMembership"
         - **Main Sections**:
           - **H1**: `id="dashboard-title"` - Title "Gym Membership Dashboard"
           - **Memberships Section**: `id="memberships-section"`
             - Contains a list of membership plans.
             - Each plan uses `<li>` containing an `<a>` link:
               - Link href generated using `url_for('plan_details', plan_id=membership.membership_id)`
               - Displays: "plan_name - $price / billing_cycle"
           - **Trainers Section**: `id="trainers-section"`
             - Contains a list of trainers, each as `<li>` including:
               - Trainer name in `<strong>`
               - Specialty text
               - Certifications joined by commas
               - Experience in years
               - Bio text
         
         - **Navigation Elements**:
           - None explicitly, navigation via links in memberships list
         
         - **Context Variables Access Pattern**:
           - `memberships` loop to access membership details
           - `trainers` loop to show trainer details
         
         ---
         
         ### Template: `memberships.html`
         - **Page Title**: "Memberships | GymMembership"
         - **Elements**:
           - H1 with `id="memberships-title"` displaying header "Membership Plans"
           - Section with `id="plans-list"` containing a `<ul>` of membership plans.
           - Each membership plan as `<li>` with anchor link:
             - `href` constructed with `url_for('plan_details', plan_id=plan.membership_id)`
             - Text: "plan_name - $price / billing_cycle"
           - Navigation link with no id:
             - Anchor back to dashboard:
               - URL generated with `url_for('dashboard')`
               - Text: "Back to Dashboard"
         
         - **Context Variables Access Pattern**:
           - Loop `memberships` to render plan list
         
         ---
         
         ### Template: `plan_details.html`
         - **Page Title**: "Plan Details | GymMembership"
         - **Elements**:
           - H1 `id="plan-title"` displaying the plan's name
           - Section `id="plan-details-section"` with paragraphs showing:
             - Price labeled and formatted as currency
             - Billing cycle
             - Features (string)
             - Max classes (integer)
           - Navigation link with no id:
             - Anchor back to memberships:
               - URL `url_for('memberships')`
               - Text "Back to Memberships"
         
         - **Context Variables Access Pattern**:
           - `plan` dict to access all displayed fields
         
         ---
         
         ## Section 3: Data File Schemas
         
         The application uses the following data files located in the root directory:
         
         ### 1. `memberships.txt`
         - Format: Each line as `membership_id|plan_name|price|billing_cycle|features|max_classes`
         - Fields:
           - `membership_id` (int): Unique membership plan identifier.
           - `plan_name` (str): Name of the membership plan.
           - `price` (float): Price for the membership.
           - `billing_cycle` (str): Billing frequency (e.g., monthly).
           - `features` (str): Description or comma-separated features.
           - `max_classes` (int): Maximum allowed classes per period.
         
         ### 2. `trainers.txt`
         - Format: Each line as `trainer_id|name|specialty|certifications(comma-separated)|experience_years|bio`
         - Fields:
           - `trainer_id` (int): Unique trainer identifier.
           - `name` (str): Trainer's full name.
           - `specialty` (str): Trainer specialty area.
           - `certifications` (str): Comma-separated list of certifications.
           - `experience_years` (int): Number of years experience.
           - `bio` (str): Short biography or description.
