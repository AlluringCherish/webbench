Integration Testing Report for BookstoreOnline

1. Functional Testing:

- Navigation and Routing:
  * Root ('/') correctly redirects to '/dashboard' without error.
  * Dashboard displays featured books (up to 4) and top 5 bestsellers based on sales count.
  * Navigation buttons on dashboard lead reliably to Catalog, Cart, and Bestsellers pages.
  * Catalog page filters books by category and search query (title, author, ISBN) with case-insensitive matching.
  * Book Details page loads correct book info, displays associated reviews sorted by newest first.
  * Adding books to cart respects stock limits and adds/updates quantities properly.
  * Cart page displays cart items with accurate quantity, price, subtotal, and total amounts.
  * Users can update quantities (enforced min=1 and capped at stock) and remove items.
  * Checkout form validates customer name, shipping address, and payment method; errors shown for invalid inputs.
  * Checkout processes orders, records order and order items with proper incremental IDs, and clears cart post successful checkout.
  * Order History page lists orders filtered by status (All, Pending, Shipped, Delivered).
  * Reviews page shows all reviews or filtered by rating with correct book titles.
  * Write Review page lists only books purchased in past orders; validates input on submit and saves new reviews.
  * Bestsellers page filters by time period, sorts and ranks bestsellers; navigation buttons function.

2. UI Consistency:

- All pages have container div with expected IDs (e.g., dashboard-page, catalog-page, book-details-page, cart-page, checkout-page, orders-page, reviews-page, write-review-page, bestsellers-page).
- All required static element IDs present (buttons, inputs, tables, forms).
- Dynamic IDs such as view-book-button-<book_id>, remove-item-button-<cart_id>, review-<review_id>, and view-order-button-<order_id> are rendered correctly with proper substitutions.
- UI handles various data sizes gracefully without layout breakage.

3. Data Integrity:

- Backend data read from text files matches frontend display for books, categories, cart contents, orders, order items, reviews, and bestsellers.
- Calculations for prices, subtotals, total amounts in cart and orders are accurate and rounded to two decimals.
- Cart quantities validated and capped at stock limits.
- Review additions generate new unique IDs and are saved correctly.
- Order and order items files updated with correct new entries on checkout.
- Cart cleared properly after order placement.

Summary:

The application meets functional, UI, and data integrity requirements across all main user workflows tested. Navigation, data filtering, CRUD operations on cart, checkout, and reviews perform correctly without errors. UI elements exist with correct IDs enabling consistent behavior and styling. No discrepancies or bugs were observed.
