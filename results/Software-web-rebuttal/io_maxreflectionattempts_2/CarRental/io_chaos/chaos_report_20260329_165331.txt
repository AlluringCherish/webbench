         | /special_requests          | special_requests        | GET, POST      | special_requests.html       | reservations: list[dict], special_requests_form_data: dict (POST)                                                  |

         **Route Details and Notes:**
         - The root route `/` redirects to `/dashboard`.
         - The dashboard shows featured vehicles and promotions.
         - Vehicle search page supports GET to show vehicles; filters by location and vehicle type via query or form.
         - Vehicle details page shows details for one vehicle.
         - Booking page includes GET to display booking form and POST to process booking data and calculate price.
         - Insurance options page retrieves insurance plans and handles insurance selection confirmation.
         - Rental history page displays rentals with optional status filter.
         - Reservations page lists current reservations.
         - Special requests page shows form and accepts POST submissions for special requests.
         - Locations page lists rental locations with filters.

         ## Section 3: Data File Schemas
         
         ### 1. vehicles.txt
         - Stores: Vehicle details for rentals.
         - Format (pipe-delimited):
           vehicle_id|make|model|vehicle_type|daily_rate|seats|transmission|fuel_type|status
         - Fields:
           - vehicle_id: int
           - make: str
           - model: str
           - vehicle_type: str (Economy, Compact, Sedan, SUV, Luxury)
           - daily_rate: float
           - seats: int
           - transmission: str (Automatic/Manual)
           - fuel_type: str
           - status: str (Available/Unavailable)
         - Examples:
           1|Toyota|Camry|Sedan|55.00|5|Automatic|Petrol|Available
           2|Honda|CR-V|SUV|75.00|7|Automatic|Petrol|Available
           3|BMW|X5|Luxury|150.00|5|Automatic|Diesel|Available
         
         ### 2. customers.txt
         - Stores: Customer details.
         - Format:
           customer_id|name|email|phone|driver_license|license_expiry
         - Fields:
           - customer_id: int
           - name: str
           - email: str
           - phone: str
           - driver_license: str
           - license_expiry: date (YYYY-MM-DD)
         - Examples:
           1|Alice Johnson|alice@example.com|555-0101|D1234567|2026-06-15
           2|Bob Williams|bob@example.com|555-0102|D2345678|2027-03-20
         
         ### 3. locations.txt
         - Stores: Rental locations information.
         - Format:
           location_id|city|address|phone|hours|available_vehicles
         - Fields:
           - location_id: int
           - city: str
           - address: str
           - phone: str
           - hours: str (24/7, Business Hours, Weekend hours like 09:00-18:00)
           - available_vehicles: int
         - Examples:
           1|New York|123 Main St, NYC|555-1000|24/7|12
           2|Los Angeles|456 Oak Ave, LA|555-2000|09:00-18:00|8
         
         ### 4. rentals.txt
         - Stores: Rental transactions.
         - Format:
           rental_id|vehicle_id|customer_id|pickup_date|dropoff_date|pickup_location|dropoff_location|total_price|status
         - Fields:
           - rental_id: int
           - vehicle_id: int
           - customer_id: int
           - pickup_date: date (YYYY-MM-DD)
           - dropoff_date: date (YYYY-MM-DD)
           - pickup_location: str
           - dropoff_location: str
           - total_price: float
           - status: str (Active, Completed, Cancelled)
         - Examples:
           1|1|1|2025-02-01|2025-02-05|New York|Los Angeles|275.00|Completed
           2|2|2|2025-02-10|2025-02-12|Los Angeles|New York|150.00|Active
         
         ### 5. insurance.txt
         - Stores: Insurance plans.
         - Format:
           insurance_id|plan_name|description|daily_cost|coverage_limit|deductible
         - Fields:
           - insurance_id: int
           - plan_name: str
           - description: str
           - daily_cost: float
           - coverage_limit: str or int (e.g., 50000 or "Unlimited")
           - deductible: float or int
         - Examples:
           1|Basic Coverage|Liability only, min coverage|5.00|50000|1000
           2|Standard Coverage|Collision and theft protection|12.00|250000|500
           3|Premium Coverage|Full comprehensive protection|18.00|Unlimited|0
         
         ### 6. reservations.txt
         - Stores: Reservations linked to rentals, with insurance and requests.
         - Format:
           reservation_id|rental_id|vehicle_id|customer_id|status|insurance_id|special_requests
         - Fields:
           - reservation_id: int
           - rental_id: int
           - vehicle_id: int
           - customer_id: int
           - status: str (Confirmed, Active, Cancelled)
           - insurance_id: int
           - special_requests: str
         - Examples:
           1|1|1|1|Confirmed|2|Driver assistance requested
           2|2|2|2|Active|1|GPS and child seat needed

         ## Section 3: Data File Schemas
         
         ### 1. vehicles.txt
         - Stores vehicle details to support vehicle search and booking.
         - Pipe-delimited format (no header):
           vehicle_id|make|model|vehicle_type|daily_rate|seats|transmission|fuel_type|status
         - Fields:
           - vehicle_id: int
           - make: str
           - model: str
           - vehicle_type: str (Economy, Compact, Sedan, SUV, Luxury)
           - daily_rate: float
           - seats: int
           - transmission: str (Automatic or Manual)
           - fuel_type: str
           - status: str (Available or Unavailable)
         - Examples:
           1|Toyota|Camry|Sedan|55.00|5|Automatic|Petrol|Available
           2|Honda|CR-V|SUV|75.00|7|Automatic|Petrol|Available
           3|BMW|X5|Luxury|150.00|5|Automatic|Diesel|Available
         
         
         ### 2. customers.txt
         - Stores customer personal information.
         - Format:
           customer_id|name|email|phone|driver_license|license_expiry
         - Fields:
           - customer_id: int
           - name: str
           - email: str
           - phone: str
           - driver_license: str
           - license_expiry: date (YYYY-MM-DD)
         - Examples:
           1|Alice Johnson|alice@example.com|555-0101|D1234567|2026-06-15
           2|Bob Williams|bob@example.com|555-0102|D2345678|2027-03-20
         
         
         ### 3. locations.txt
         - Stores rental location info with availability.
         - Format:
           location_id|city|address|phone|hours|available_vehicles
         - Fields:
           - location_id: int
           - city: str
           - address: str
           - phone: str
           - hours: str (24/7, Business Hours, Weekend, or hours range)
           - available_vehicles: int
         - Examples:
           1|New York|123 Main St, NYC|555-1000|24/7|12
           2|Los Angeles|456 Oak Ave, LA|555-2000|09:00-18:00|8
         
         
         ### 4. rentals.txt
         - Stores rental transaction records.
         - Format:
           rental_id|vehicle_id|customer_id|pickup_date|dropoff_date|pickup_location|dropoff_location|total_price|status
         - Fields:
           - rental_id: int
           - vehicle_id: int
           - customer_id: int
           - pickup_date: date (YYYY-MM-DD)
           - dropoff_date: date (YYYY-MM-DD)
           - pickup_location: str
           - dropoff_location: str
           - total_price: float
           - status: str (Active, Completed, Cancelled)
         - Examples:
           1|1|1|2025-02-01|2025-02-05|New York|Los Angeles|275.00|Completed
           2|2|2|2025-02-10|2025-02-12|Los Angeles|New York|150.00|Active
         
         
         ### 5. insurance.txt
         - Stores insurance plan details.
         - Format:
           insurance_id|plan_name|description|daily_cost|coverage_limit|deductible
         - Fields:
           - insurance_id: int
           - plan_name: str
           - description: str
           - daily_cost: float
           - coverage_limit: str/int (e.g., 50000 or "Unlimited")
           - deductible: float/int
         - Examples:
           1|Basic Coverage|Liability only, min coverage|5.00|50000|1000
           2|Standard Coverage|Collision and theft protection|12.00|250000|500
           3|Premium Coverage|Full comprehensive protection|18.00|Unlimited|0
         
         
         ### 6. reservations.txt
         - Stores reservation details and special requests.
         - Format:
           reservation_id|rental_id|vehicle_id|customer_id|status|insurance_id|special_requests
         - Fields:
           - reservation_id: int
           - rental_id: int
           - vehicle_id: int
           - customer_id: int
           - status: str (Confirmed, Active, Cancelled)
           - insurance_id: int
           - special_requests: str
         - Examples:
           1|1|1|1|Confirmed|2|Driver assistance requested
           2|2|2|2|Active|1|GPS and child seat needed

         ## Section 3: Data File Schemas
         
         ### 1. vehicles.txt
         - Stores vehicle details to support vehicle search and booking.
         - Pipe-delimited format (no header):
           vehicle_id|make|model|vehicle_type|daily_rate|seats|transmission|fuel_type|status
         - Fields:
           - vehicle_id: int
           - make: str
           - model: str
           - vehicle_type: str (Economy, Compact, Sedan, SUV, Luxury)
           - daily_rate: float
           - seats: int
           - transmission: str (Automatic or Manual)
           - fuel_type: str
           - status: str (Available or Unavailable)
         - Examples:
           1|Toyota|Camry|Sedan|55.00|5|Automatic|Petrol|Available
           2|Honda|CR-V|SUV|75.00|7|Automatic|Petrol|Available
           3|BMW|X5|Luxury|150.00|5|Automatic|Diesel|Available
         
         
         ### 2. customers.txt
         - Stores customer personal information.
         - Format:
           customer_id|name|email|phone|driver_license|license_expiry
         - Fields:
           - customer_id: int
           - name: str
           - email: str
           - phone: str
           - driver_license: str
           - license_expiry: date (YYYY-MM-DD)
         - Examples:
           1|Alice Johnson|alice@example.com|555-0101|D1234567|2026-06-15
           2|Bob Williams|bob@example.com|555-0102|D2345678|2027-03-20
         
         
         ### 3. locations.txt
         - Stores rental location info with availability.
         - Format:
           location_id|city|address|phone|hours|available_vehicles
         - Fields:
           - location_id: int
           - city: str
           - address: str
           - phone: str
