Integration Test Report for RestaurantReservation Application

1. Dashboard Page
- Loads successfully with elements including welcome message, buttons (make reservation, view menu, refresh dashboard, my reservations, my reviews, waitlist, profile).
- Displays featured dishes correctly with name and price formatted with two decimals.
- Lists upcoming reservations for logged-in user with date, time, party size, status.

2. Menu Page
- Loads all menu items with category, description, price formatted correctly.
- Each dish has a "View Details" button linking to correct dish detail page.

3. Dish Details Page
- Displays dish info: name, category, description, ingredients (comma separated), dietary info, average rating (one decimal).
- Back to menu button navigates correctly.
- If invalid or unknown dish_id is requested, redirects to menu page.

4. Make Reservation Page
- Form fields: Guest Name (required), Party Size (1-10), Reservation Date (required).
- Submit button sends POST to /submit_reservation.
- Back to dashboard button works.

5. Submit Reservation
- Validates guest name, party size, reservation date.
- Assigns reservation ID incrementally.
- Uses logged-in user's phone and email from users.txt.
- Saves reservation with status Upcoming, default time 19:00.
- Redirects to My Reservations page.

6. My Reservations Page
- Lists all reservations of logged-in user.
- Sorts reservations descending by date/time.
- Shows cancel button for "Upcoming" reservations with POST form.
- Cancel button form submits to /cancel_reservation/<id>.
- Cancelled status shown correctly and disables cancel.
- Back to dashboard button present.

7. Cancel Reservation
- Changes reservation status to "Cancelled" in data file after POST.
- Redirects to My Reservations.

8. Waitlist Page
- Displays user's position in waitlist if active, else message not on waitlist.
- Allows joining waitlist via POST form with party size selection 1-10.
- Joining adds entry with unique ID and current timestamp.
- Back to dashboard button present.

9. Join Waitlist
- Prevents multiple active entries per user.
- Adds correctly and redirects.

10. My Reviews Page
- Lists logged-in user's reviews with dish name, rating, review text.
- Buttons for writing new review and back to dashboard present and functional.

11. Write Review Page
- Form with dish dropdown, rating selection 1-5, and review text textarea.
- Submits POST to /submit_review.
- Back to reviews button present.

12. Submit Review
- Validates dish ID and rating.
- Assigns new review ID.
- Saves review with current date.
- Redirects to My Reviews page.

13. Profile Page
- Displays username and current email.
- Form to update email POSTs to /update_profile.
- Back to dashboard button present.

14. Update Profile
- Updates user's email in users.txt.
- Redirects to profile page.

Additional Notes:
- All templates contain required IDs and button IDs for UI elements.
- Dynamic button IDs for dish detail views and reservation cancels are present.
- Form actions and methods match Flask routes.

Known Caveats / Improvements:
- Reservation time in make reservation is fixed at 19:00 in backend since form doesn't allow input.
- Users not found in users.txt have limited profile functionality.
- No authentication implemented; user is hardcoded.

End of Report
