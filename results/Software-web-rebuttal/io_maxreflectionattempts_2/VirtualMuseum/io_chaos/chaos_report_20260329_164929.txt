         # VirtualMuseum Design Specification Document

         |---|---|---|---|---|
         | / | root_redirect | GET | None (redirect to /dashboard) | None |
         | /dashboard | dashboard | GET | dashboard.html | exhibition_summary (dict: total_exhibitions:int, active_exhibitions:int) |

         | /exhibitions | exhibitions | GET, POST | exhibitions.html | exhibitions (list of dict), exhibition_types (list of str), selected_type (str, optional) |

            - Method: GET

            - Template: `artifact_catalog.html`

              - `filter_applied` (bool): Whether a filter/search was applied

            - Function: `exhibitions`

            - Method: GET
            - Template: `exhibition_details.html`

            - Context Variables:

            - Function: `audio_guides`
            - Methods: GET (view guides), POST (apply language filter)

              - `languages` (list of str): ['English', 'Spanish', 'French']

            - Path: `/artifact_catalog`
            - Function Name: `artifact_catalog`
            - HTTP Methods: GET (load page), POST (apply search/filter)
            - Template: `artifact_catalog.html`
            - Context Variables:
              - `artifacts` (list of dict): List of artifact objects
              - `search_query` (str, optional): Search input from user
              - `filter_applied` (bool): Whether any filter/search applied
         
         4. **Exhibitions Page**
            - Path: `/exhibitions`
            - Function Name: `exhibitions`
            - HTTP Methods: GET (load page), POST (apply exhibition type filter)
            - Template: `exhibitions.html`
            - Context Variables:
              - `exhibitions` (list of dict): List of exhibitions
              - `exhibition_types` (list of str): ['Permanent', 'Temporary', 'Virtual']
              - `selected_type` (str, optional): Currently selected filter
         
         5. **Exhibition Details Page**
            - Path: `/exhibition/<int:exhibition_id>`
            - Function Name: `exhibition_details`
            - HTTP Method: GET
            - Template: `exhibition_details.html`
            - Context Variables:
              - `exhibition` (dict): Exhibition data
              - `artifacts` (list of dict): Artifacts contained in this exhibition
         
         6. **Visitor Tickets Page**
            - Path: `/visitor_tickets`
            - Function Name: `visitor_tickets`
            - HTTP Methods: GET (view tickets), POST (purchase tickets)
            - Template: `visitor_tickets.html`
            - Context Variables:
              - `ticket_types` (list of str): ['Standard', 'Student', 'Senior', 'Family', 'VIP']
              - `purchased_tickets` (list of dict): Tickets by current user
              - `purchase_message` (str, optional): Confirmation or error message
         
         7. **Virtual Events Page**
            - Path: `/virtual_events`
            - Function Name: `virtual_events`
            - HTTP Methods: GET (view events), POST (register/cancel registration)
            - Template: `virtual_events.html`
            - Context Variables:
              - `events` (list of dict): List of all events
              - `user_registrations` (list of dict): Registrations by user
              - `registration_message` (str, optional): Feedback message
         
         8. **Audio Guides Page**
            - Path: `/audio_guides`
            - Function Name: `audio_guides`
            - HTTP Methods: GET (view guides), POST (apply language filter)
            - Template: `audio_guides.html`
            - Context Variables:
              - `audio_guides` (list of dict): List of audio guides
              - `languages` (list of str): ['English', 'Spanish', 'French']
              - `selected_language` (str, optional): Current language filter

         ## Section 3: Data File Schemas
         
         ### 1. users.txt
         - Path: data/users.txt
         - Fields (pipe-delimited):
           - username
         - Purpose: Store usernames for login/authentication
         - Example rows:
           curator_john
           visitor_mary
           curator_sarah
         
         ### 2. galleries.txt
         - Path: data/galleries.txt
         - Fields:
           gallery_id|gallery_name|floor|capacity|theme|status
         - Purpose: Contains information about galleries and their status
         - Example rows:
           1|Ancient Civilizations Hall|1|50|Ancient|Open
           2|Modern Art Wing|2|30|Modern|Open
           3|Science Discovery Center|3|40|Science|Renovation
         
         ### 3. exhibitions.txt
         - Path: data/exhibitions.txt
         - Fields:
           exhibition_id|title|description|gallery_id|exhibition_type|start_date|end_date|curator_name|created_by
         - Purpose: Stores exhibition metadata
         - Example rows:
           1|Egyptian Treasures|Ancient artifacts from Egypt|1|Permanent|2024-01-01|2025-12-31|Dr. Smith|curator_john
           2|Pop Art Revolution|Contemporary pop art collection|2|Temporary|2024-11-01|2024-12-31|Lisa Chen|curator_sarah
           3|Virtual Space Exploration|Online space exhibition|2|Virtual|2024-10-01|2025-03-31|Mark Johnson|curator_john
         
         ### 4. artifacts.txt
         - Path: data/artifacts.txt
         - Fields:
           artifact_id|artifact_name|period|origin|description|exhibition_id|storage_location|acquisition_date|added_by
         - Purpose: Stores artifact details and metadata
         - Example rows:
           1|Golden Mask of Tutankhamun|Ancient|Egypt|Famous golden funeral mask|1|A-101|2020-05-15|curator_john
           2|Warhol Soup Cans|Contemporary|USA|Iconic pop art piece|2|B-205|2023-08-20|curator_sarah
           3|Roman Amphora|Ancient|Italy|Ancient storage vessel|1|A-103|2019-11-10|curator_john
         
         ### 5. audioguides.txt
         - Path: data/audioguides.txt
         - Fields:
           guide_id|exhibit_number|title|language|duration|script|narrator|created_by
         - Purpose: Audio guide information for exhibits
         - Example rows:
           1|101|Introduction to Egyptian Art|English|5|Welcome to the Egyptian exhibition...|James Brown|curator_john
           2|101|Introducción al Arte Egipcio|Spanish|5|Bienvenido a la exhibición egipcia...|Maria Garcia|curator_john
           3|205|Pop Art Explained|English|7|Pop art emerged in the 1950s...|Sarah Williams|curator_sarah
         
         ### 6. tickets.txt
         - Path: data/tickets.txt
         - Fields:
           ticket_id|username|ticket_type|visit_date|visit_time|number_of_tickets|price|visitor_name|visitor_email|purchase_date
         - Purpose: Visitor ticket purchases and details
         - Example rows:
           1|visitor_mary|Standard|2024-11-25|11:00 AM|2|30|Mary Johnson|mary@email.com|2024-11-20
           2|visitor_mary|VIP|2024-12-01|1:00 PM|1|50|Mary Johnson|mary@email.com|2024-11-20
           3|visitor_tom|Student|2024-11-22|9:00 AM|1|10|Tom Lee|tom@email.com|2024-11-18
         
         ### 7. events.txt
         - Path: data/events.txt
         - Fields:
           event_id|title|date|time|event_type|speaker|capacity|description|created_by
         - Purpose: Virtual event details
         - Example rows:
           1|Ancient Egypt Webinar|2024-12-05|2:00 PM|Webinar|Dr. Smith|100|Deep dive into Egyptian culture|curator_john
           2|Meet the Artist: Pop Art|2024-11-30|6:00 PM|Artist Talk|Andy Williams|50|Discussion with contemporary artist|curator_sarah
           3|Virtual Museum Tour|2024-12-10|10:00 AM|Virtual Tour|Lisa Chen|200|Guided tour of entire museum|curator_john
         
         ### 8. event_registrations.txt
         - Path: data/event_registrations.txt
         - Fields:
           registration_id|event_id|username|registration_date
         - Purpose: User registrations for virtual events
         - Example rows:
           1|1|visitor_mary|2024-11-20
           2|2|visitor_mary|2024-11-21
           3|1|visitor_tom|2024-11-19
         
         ### 9. collection_logs.txt
         - Path: data/collection_logs.txt
         - Fields:
           log_id|artifact_id|activity_type|date|notes|condition|curator
         - Purpose: Logs of artifact collection activities, maintenance and conditions
         - Example rows:
           1|1|Inspection|2024-11-15|Routine quarterly inspection|Excellent|curator_john
           2|1|Cleaning|2024-11-16|Professional cleaning performed|Excellent|curator_john
           3|2|Photography|2024-10-20|High-resolution images taken|Good|curator_sarah
           4|3|Restoration|2024-09-10|Minor crack repair completed|Good|curator_john

         ## Section 1: Flask Routes Specification
         
         | Route Path | Function Name | HTTP Method | Template Filename | Context Variables |
         |---|---|---|---|---|
         | / | root_redirect | GET | None (redirect to /dashboard) | None |
         | /dashboard | dashboard | GET | dashboard.html | exhibition_summary (dict: total_exhibitions:int, active_exhibitions:int) |
         | /artifact_catalog | artifact_catalog | GET, POST | artifact_catalog.html | artifacts (list of dict), search_query (str, optional), filter_applied (bool) |
         | /exhibitions | exhibitions | GET, POST | exhibitions.html | exhibitions (list of dict), exhibition_types (list of str), selected_type (str, optional) |
         | /exhibition/<int:exhibition_id> | exhibition_details | GET | exhibition_details.html | exhibition (dict), artifacts (list of dict) |
         | /visitor_tickets | visitor_tickets | GET, POST | visitor_tickets.html | ticket_types (list of str), purchased_tickets (list of dict), purchase_message (str, optional) |
         | /virtual_events | virtual_events | GET, POST | virtual_events.html | events (list of dict), registered_events (list of int: registration_id), user_registrations (list of dict), registration_message (str, optional) |
         | /audio_guides | audio_guides | GET, POST | audio_guides.html | audio_guides (list of dict), languages (list of str), selected_language (str, optional) |
         
         Details per route:
         
         1. **Root Route**
            - Path: `/`
            - Function: `root_redirect`
            - Method: GET
            - Redirects to `/dashboard`
         
         2. **Dashboard Route**
            - Path: `/dashboard`
            - Function: `dashboard`
            - Method: GET
            - Template: `dashboard.html`
            - Context Variables:
              - `exhibition_summary` (dict):
                - `total_exhibitions` (int): Total number of exhibitions
                - `active_exhibitions` (int): Count of exhibitions currently active
         
         3. **Artifact Catalog**
            - Path: `/artifact_catalog`
            - Function: `artifact_catalog`
            - Methods: GET (load page), POST (apply search/filter)
            - Template: `artifact_catalog.html`
            - Context Variables:
              - `artifacts` (list of dict): List of artifact records with keys matching artifact fields
              - `search_query` (str, optional): Search string submitted by user
              - `filter_applied` (bool): Whether a filter/search was applied
         
         4. **Exhibitions**
            - Path: `/exhibitions`
            - Function: `exhibitions`
            - Methods: GET (load page), POST (apply exhibition type filter)
            - Template: `exhibitions.html`
            - Context Variables:
              - `exhibitions` (list of dict): List of exhibition records
              - `exhibition_types` (list of str): ['Permanent', 'Temporary', 'Virtual']
              - `selected_type` (str, optional): Selected filter type
         
         5. **Exhibition Details**
            - Path: `/exhibition/<int:exhibition_id>`
            - Function: `exhibition_details`
            - Method: GET
            - Template: `exhibition_details.html`
            - Context Variables:
              - `exhibition` (dict): Exhibition record
              - `artifacts` (list of dict): Artifact records associated with this exhibition
         
         6. **Visitor Tickets**
            - Path: `/visitor_tickets`
            - Function: `visitor_tickets`
            - Methods: GET (show purchased tickets), POST (purchase new tickets)
            - Template: `visitor_tickets.html`
            - Context Variables:
              - `ticket_types` (list of str): ['Standard', 'Student', 'Senior', 'Family', 'VIP']
              - `purchased_tickets` (list of dict): Tickets purchased by the logged-in user
              - `purchase_message` (str, optional): Confirmation or error message related to purchase
         
         7. **Virtual Events**
            - Path: `/virtual_events`
            - Function: `virtual_events`
            - Methods: GET (view events), POST (register or cancel registration)
            - Template: `virtual_events.html`
            - Context Variables:
              - `events` (list of dict): All event records
              - `user_registrations` (list of dict): Registrations by the logged-in user, each with registration_id and event_id
              - `registration_message` (str, optional): Message after registration or cancellation
         
         8. **Audio Guides**
            - Path: `/audio_guides`
            - Function: `audio_guides`
            - Methods: GET (view guides), POST (apply language filter)
            - Template: `audio_guides.html`
            - Context Variables:
              - `audio_guides` (list of dict): Audio guide records
              - `languages` (list of str): ['English', 'Spanish', 'French']
              - `selected_language` (str, optional): Selected language filter

         ## Section 3: Data File Schemas
         
         1. users.txt
         - Path: data/users.txt
         - Fields (pipe delimited):
           - username
         - Purpose: Stores usernames for authentication
         - Examples:
           curator_john
           visitor_mary
           curator_sarah
         
         2. galleries.txt
         - Path: data/galleries.txt
         - Fields:
           gallery_id|gallery_name|floor|capacity|theme|status
         - Purpose: Stores gallery details
         - Examples:
           1|Ancient Civilizations Hall|1|50|Ancient|Open
           2|Modern Art Wing|2|30|Modern|Open
           3|Science Discovery Center|3|40|Science|Renovation
         
         3. exhibitions.txt
         - Path: data/exhibitions.txt
         - Fields:
           exhibition_id|title|description|gallery_id|exhibition_type|start_date|end_date|curator_name|created_by
         - Purpose: Store exhibitions metadata
         - Examples:
           1|Egyptian Treasures|Ancient artifacts from Egypt|1|Permanent|2024-01-01|2025-12-31|Dr. Smith|curator_john
           2|Pop Art Revolution|Contemporary pop art collection|2|Temporary|2024-11-01|2024-12-31|Lisa Chen|curator_sarah
           3|Virtual Space Exploration|Online space exhibition|2|Virtual|2024-10-01|2025-03-31|Mark Johnson|curator_john
         
         4. artifacts.txt
         - Path: data/artifacts.txt
         - Fields:
           artifact_id|artifact_name|period|origin|description|exhibition_id|storage_location|acquisition_date|added_by
         - Purpose: Store artifact details
         - Examples:
           1|Golden Mask of Tutankhamun|Ancient|Egypt|Famous golden funeral mask|1|A-101|2020-05-15|curator_john
           2|Warhol Soup Cans|Contemporary|USA|Iconic pop art piece|2|B-205|2023-08-20|curator_sarah
           3|Roman Amphora|Ancient|Italy|Ancient storage vessel|1|A-103|2019-11-10|curator_john
         
         5. audioguides.txt
         - Path: data/audioguides.txt
         - Fields:
           guide_id|exhibit_number|title|language|duration|script|narrator|created_by
         - Purpose: Store audio guide info
         - Examples:
           1|101|Introduction to Egyptian Art|English|5|Welcome to the Egyptian exhibition...|James Brown|curator_john
           2|101|Introducción al Arte Egipcio|Spanish|5|Bienvenido a la exhibición egipcia...|Maria Garcia|curator_john
           3|205|Pop Art Explained|English|7|Pop art emerged in the 1950s...|Sarah Williams|curator_sarah
         
         6. tickets.txt
         - Path: data/tickets.txt
         - Fields:
           ticket_id|username|ticket_type|visit_date|visit_time|number_of_tickets|price|visitor_name|visitor_email|purchase_date
         - Purpose: Store visitor ticket purchases
         - Examples:
           1|visitor_mary|Standard|2024-11-25|11:00 AM|2|30|Mary Johnson|mary@email.com|2024-11-20
           2|visitor_mary|VIP|2024-12-01|1:00 PM|1|50|Mary Johnson|mary@email.com|2024-11-20
           3|visitor_tom|Student|2024-11-22|9:00 AM|1|10|Tom Lee|tom@email.com|2024-11-18
         
         7. events.txt
         - Path: data/events.txt
         - Fields:
           event_id|title|date|time|event_type|speaker|capacity|description|created_by
         - Purpose: Store virtual event details
         - Examples:
           1|Ancient Egypt Webinar|2024-12-05|2:00 PM|Webinar|Dr. Smith|100|Deep dive into Egyptian culture|curator_john
           2|Meet the Artist: Pop Art|2024-11-30|6:00 PM|Artist Talk|Andy Williams|50|Discussion with contemporary artist|curator_sarah
           3|Virtual Museum Tour|2024-12-10|10:00 AM|Virtual Tour|Lisa Chen|200|Guided tour of entire museum|curator_john
         
         8. event_registrations.txt
         - Path: data/event_registrations.txt
         - Fields:
           registration_id|event_id|username|registration_date
         - Purpose: Track user event registrations
         - Examples:

         ### Route Details:
         
         1. **Root Route**
            - Path: `/`
            - Function Name: `root_redirect`
            - HTTP Method: GET
            - Behavior: Redirects to `/dashboard`
            - Template: None
            - Context Variables: None
         
         2. **Dashboard Page**
            - Path: `/dashboard`
            - Function Name: `dashboard`
            - HTTP Method: GET
            - Template: `dashboard.html`
            - Context Variables:
              - `exhibition_summary`: dict with keys:
                - `total_exhibitions` (int)
                - `active_exhibitions` (int)
         
         3. **Artifact Catalog Page**
            - Path: `/artifact_catalog`
            - Function Name: `artifact_catalog`
            - HTTP Methods: GET (load page), POST (apply search/filter)
            - Template: `artifact_catalog.html`
            - Context Variables:
              - `artifacts` (list of dict): List of artifact objects
              - `search_query` (str, optional): Search input from user
              - `filter_applied` (bool): Whether any filter/search applied
         
         4. **Exhibitions Page**
            - Path: `/exhibitions`
            - Function Name: `exhibitions`
            - HTTP Methods: GET (load page), POST (apply exhibition type filter)
            - Template: `exhibitions.html`
            - Context Variables:
              - `exhibitions` (list of dict): List of exhibitions
              - `exhibition_types` (list of str): ['Permanent', 'Temporary', 'Virtual']
              - `selected_type` (str, optional): Currently selected filter
         
         5. **Exhibition Details Page**
            - Path: `/exhibition/<int:exhibition_id>`
            - Function Name: `exhibition_details`
            - HTTP Method: GET
            - Template: `exhibition_details.html`
            - Context Variables:
              - `exhibition` (dict): Exhibition data
              - `artifacts` (list of dict): Artifacts contained in this exhibition
         
         6. **Visitor Tickets Page**
            - Path: `/visitor_tickets`
            - Function Name: `visitor_tickets`
            - HTTP Methods: GET (view tickets), POST (purchase tickets)
            - Template: `visitor_tickets.html`
            - Context Variables:
              - `ticket_types` (list of str): ['Standard', 'Student', 'Senior', 'Family', 'VIP']
              - `purchased_tickets` (list of dict): Tickets by current user
              - `purchase_message` (str, optional): Confirmation or error message
         
         7. **Virtual Events Page**
            - Path: `/virtual_events`
            - Function Name: `virtual_events`
            - HTTP Methods: GET (view events), POST (register/cancel registration)
            - Template: `virtual_events.html`
            - Context Variables:
              - `events` (list of dict): List of all events
              - `user_registrations` (list of dict): Registrations by user
              - `registration_message` (str, optional): Feedback message
         
         8. **Audio Guides Page**
            - Path: `/audio_guides`
            - Function Name: `audio_guides`
            - HTTP Methods: GET (view guides), POST (apply language filter)
            - Template: `audio_guides.html`
            - Context Variables:
              - `audio_guides` (list of dict): List of audio guides
              - `languages` (list of str): ['English', 'Spanish', 'French']
              - `selected_language` (str, optional): Current language filter
