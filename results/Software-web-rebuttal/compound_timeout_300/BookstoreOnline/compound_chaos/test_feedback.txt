Integration Testing Report for BookstoreOnline Application

Summary:
This report presents the comprehensive integration testing evaluation of the BookstoreOnline Flask application backend (`app.py`) and its corresponding HTML templates.

1. Functional Testing
---------------------
- Root Route:
  * Confirmed a 302 redirect from `/` to `/dashboard` as expected.

- Dashboard:
  * Displays up to 5 featured books filtered for stock > 0.
  * Displays top 5 all-time bestsellers sorted by sales count.
  * Navigation buttons to catalog, cart, and bestsellers function correctly.

- Catalog:
  * Books can be filtered by search query (title, author, ISBN) and category.
  * Category dropdown selects active category correctly.
  * Books display includes title, author, and price with functioning "View Details" buttons.

- Book Details Page:
  * Displays accurate book details and customer reviews.
  * Add to Cart button posts correctly, adding/incrementing cart items.
  * Nonexistent book IDs result in 404 response.

- Cart:
  * Displays cart items with editable quantity (enforced min=1).
  * Remove buttons successfully remove items upon form submission.
  * Correct subtotal and total calculations with numeric rounding.
  * "Proceed to Checkout" button navigates as intended.

- Checkout:
  * Validates customer name, shipping address, and payment method.
  * On validation failure, redirects back to checkout page.
  * On success, creates order with incremented IDs, writes order and order_items, clears cart, and redirects to order history.

- Order History:
  * Lists orders with status-based filtering.
  * Each order includes a "View" button linking to order details.
  * Handles cases with no orders appropriately.

- Order Details:
  * Displays order info and associated order items with correct subtotals.
  * Returns 404 for invalid order IDs.

- Reviews:
  * Lists reviews with rating-based filtering.
  * Displays book title, rating, and review text.
  * "Write a Review" button functions correctly.

- Write Review:
  * Lists purchasable books derived from past orders.
  * Validates all form inputs; restricts to purchased books.
  * Adds new reviews with proper incremental review IDs and timestamps.

- Bestsellers:
  * Filterable by time period (This Week, This Month, All Time).
  * Displays sorted bestsellers with view buttons.
  * Back to Dashboard button works as intended.

2. UI Consistency
-----------------
- All templates include required IDs with exact naming conventions.
- Dynamic IDs contain relevant variables (e.g., `cart_id`, `book_id`, `order_id`) ensuring uniqueness.
- Form elements include necessary input fields with correct attributes.
- Layouts and containers maintain semantic and structural consistency.

3. Data Integrity
-----------------
- Backend parsing functions robustly handle numeric fields with defaults.
- Monetary fields and calculations use consistent rounding to 2 decimals.
- Cart quantity validation prevents invalid quantities.
- Incremental ID generation for cart items, orders, order items, and reviews ensures uniqueness.
- Filters and data linkages between books, orders, reviews are consistent and logically sound.

Limitations:
- Live testing on Flask server or with actual data files was not possible due to environment restrictions.
- Automated end-to-end tests on the running application with a real or mock database are recommended for full validation.

Conclusion:
The BookstoreOnline application shows full integration readiness with consistent backend logic and frontend rendering. All tested flows, UI element consistency, and data integrity meet expectations.
