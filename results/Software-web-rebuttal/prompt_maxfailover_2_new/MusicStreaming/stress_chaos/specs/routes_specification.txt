# Flask Routes Specification for MusicStreaming Application

1. Root Route
- URL: /
- Method: GET
- Function: redirect_to_dashboard
- Behavior: Redirects to /dashboard

2. Dashboard
- URL: /dashboard
- Method: GET
- Function: dashboard
- Template: dashboard.html
- Context Variables:
  - featured_songs: list of featured song dicts (each song dict has id, title, artist_name, duration)
  - featured_playlists: list of playlists dicts (id, title, total_songs)
  - trending_songs: list of trending songs dicts

3. Song Catalog
- URL: /songs
- Method: GET
- Function: song_catalog
- Template: songs.html
- Context Variables:
  - songs: list of songs dicts (id, title, album, duration, genre)
  - genres: list of genre dicts (id, genre_name)
  - current_genre_filter: genre_id or None
  - search_query: string or empty

4. Song Details
- URL: /songs/<int:song_id>
- Method: GET
- Function: song_details
- Template: song_detail.html
- Context Variables:
  - song: dict with song details (id, title, artist_name, duration, album, release_date, genre)

5. Playlist Page
- URL: /playlists
- Method: GET
- Function: playlists
- Template: playlists.html
- Context Variables:
  - playlists: list of playlist dicts (id, title, description, total_songs)

6. Playlist Details
- URL: /playlists/<int:playlist_id>
- Method: GET
- Function: playlist_details
- Template: playlist_detail.html
- Context Variables:
  - playlist: dict (id, title, description, creation_date)
  - songs: list of song dicts in playlist (id, title, artist_name, duration)

7. Create Playlist
- URL: /playlists/create
- Method: GET, POST
- Function: create_playlist
- Template: create_playlist.html (GET)
- Context Variables on GET: None
- On POST: process form data to create new playlist

8. Albums List
- URL: /albums
- Method: GET
- Function: albums_list
- Template: albums.html
- Context Variables:
  - albums: list of album dicts (id, title, artist_name, release_year, genre)
  - years: list of years (int) for filtering
  - current_year_filter: year or None

9. Album Details
- URL: /albums/<int:album_id>
- Method: GET
- Function: album_details
- Template: album_detail.html
- Context Variables:
  - album: dict (id, title, artist_name, release_year, genre)
  - songs: list of song dicts (id, title, duration)

10. Artists List
- URL: /artists
- Method: GET
- Function: artists_list
- Template: artists.html
- Context Variables:
  - artists: list of artist dicts (id, name, genre, country, active_since)
  - search_query: string or empty
  - sort_option: string (Name or Genre)

11. Artist Profile
- URL: /artists/<int:artist_id>
- Method: GET
- Function: artist_profile
- Template: artist_profile.html
- Context Variables:
  - artist: dict (id, name, genre, country, active_since)
  - albums: list of albums by artist
  - songs: list of songs by artist

12. Genre Page
- URL: /genres
- Method: GET
- Function: genres_page
- Template: genres.html
- Context Variables:
  - genres: list of genre dicts (id, name, description)

13. Genre Details
- URL: /genres/<int:genre_id>
- Method: GET
- Function: genre_details
- Template: genre_detail.html
- Context Variables:
  - genre: dict (id, name, description)
  - artists: list of artists in genre
  - songs: list of songs in genre

Navigation:
- Root redirects to /dashboard
- Dashboard links to /songs, /playlists, /albums, /artists, /genres
- Songs catalog has filtering and searching capabilities
- Playlists allow creation and viewing details
- Albums and Artists have list and detail views

Dynamic Parameters:
- song_id: integer
- playlist_id: integer
- album_id: integer
- artist_id: integer
- genre_id: integer

All templates use consistent IDs as per UI specification.
