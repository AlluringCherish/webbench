1. Backend Route Methods and Status Codes:
- '/' redirects to '/dashboard' via redirect and url_for correctly (GET).
- '/dashboard' returns 200 and renders 'dashboard.html' with featured_songs, trending_artists context.
- '/songs' (GET) renders song_catalog.html with songs list and genres.
- '/songs/search' (POST) accepts form data 'search_text', 'genre_filter', filters songs accordingly. Returns filtered songs and genres.
- '/songs/<song_id>' (GET) checks if song_id exists; returns 404 if not, else renders 'song_details.html' with full song context.
- '/playlists' (GET) renders playlists.html with list of playlists.
- '/playlists/create' (GET) shows create_playlist.html form.
- '/playlists/create' (POST) validates 'title' (non-empty), reads existing playlists, creates new playlist with unique id and current date, appends, writes back, then redirects to playlist details page.
- '/playlists/<playlist_id>' (GET) looks up playlist by id; 404 if not found; fetches songs in playlist; renders playlist_details.html.
- '/playlists/<playlist_id>/remove_song/<song_id>' (POST) removes a song from a playlist, updates total songs count, writes changes.
- '/playlists/<playlist_id>/delete' (POST) deletes entire playlist and associated songs.
- '/albums' (GET) renders albums.html with albums and artists info.
- '/albums/search' (POST) filters albums by search text.
- '/albums/<album_id>' (GET) validates album, 404 if missing; fetches songs; renders album_details.html.
- '/albums/<album_id>/add_to_playlist' (POST) adds all album's songs to a playlist.
- '/artists' (GET) shows all artists.
- '/artists/search' (POST) filters artists by search text.
- '/artists/<artist_id>' (GET) shows artist profile; 404 if not found; lists songs by artist.
- '/genres' (GET) lists music genres.
- '/genres/select' (POST) filters songs and artists by selected genre; shows genre page with lists.
